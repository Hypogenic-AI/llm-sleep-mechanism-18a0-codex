Late-night language-model interactions raise a concrete product and safety question: when a user asks for safe help at 2:43 AM, should an assistant continue helping, suggest rest, or disengage? Prior work studies refusal, over-refusal, sycophancy, personalization, and vulnerable conversations, but it does not directly test assistant-initiated ``go to sleep'' behavior as a behavioral pattern. We introduce \methodname, a small controlled benchmark for measuring \bedtimenudges across matched late-night prompts. We ran 108 real OpenAI model generations in a factorial design crossing model, user profile, context length, task urgency, and paraphrase. We labeled each response with both keyword rules and an LLM judge, separating soft answer-plus-nudge behavior from hard disengagement and refusal. Bedtime nudges were rare: only 2/108 synthetic responses (1.85\%) were labeled as nudges, both from \gptfourmini, both in the fatigued profile, and both still answered the requested task. We observed zero hard disengagements, zero task refusals, and a 100\% answered-request rate. A bounded \wildchat sleep-phrase audit found 5/25 soft nudges and no hard disengagements, showing that phrase matching alone overstates the behavior. These results do not support a unified bedtime-nudge response in the tested setting; they instead suggest a rare, model- and context-conditioned surface behavior.
